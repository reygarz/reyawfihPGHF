Основы HTML

Что такое HTML?

HTML (HyperText Markup Language) — это стандартный язык разметки документов в Интернете. Он используется для структурирования веб-страницы и её содержимого.

Ключевые концепции:

  • Теги: Основные строительные блоки (например, <p>, <div>, <a>).
  • Атрибуты: Предоставляют дополнительную информацию о тегах (например, href в <a>).
  • DOM: (Document Object Model) — древовидное представление HTML-документа, с которым работает браузер.
  • Семантическая разметка: Использование тегов по их смысловому назначению (<header>, <nav>, <article>), что улучшает SEO и доступность.
  • Формы: Теги <form>, <input>, <button> для сбора данных пользователя.

Пример базовой структуры:

<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Моя страница</title>
</head>
<body>
  <header>
    <h1>Заголовок сайта</h1>
  </header>
  <main>
    <p>Это параграф текста.</p>
    <a href="https1://example.com">Это ссылка</a>
  </main>
</body>
</html>

Основы CSS

Что такое CSS?

CSS (Cascading Style Sheets) — это язык стилей, используемый для описания внешнего вида HTML-документа. CSS определяет, как элементы должны отображаться на экране, на бумаге или в других медиа.

Ключевые концепции:

  • Селекторы: Правила, которые "находят" HTML-элементы для стилизации (по тегу, классу .class, ID #id).
  • Свойства и значения: Пары, определяющие стиль (например, color: red;).
  • Box Model (Блочная модель): Концепция, описывающая каждый элемент как прямоугольный блок с margin (отступ), border (граница), padding (поле) и content (содержимое).
  • Flexbox: Модель компоновки для создания сложных и гибких макетов в одном измерении.
  • Grid Layout: Модель компоновки для создания двумерных (строки и столбцы) макетов.
  • Медиа-запросы: (@media) Используются для применения разных стилей в зависимости от характеристик устройства (например, ширины экрана), что обеспечивает адаптивность.

Пример CSS-правила:

/* Этот селектор выберет все элементы с классом "nav-button" */
.nav-button {
  background-color: #333; /* Свойство: цвет фона */
  color: #FFF;            /* Свойство: цвет текста */
  padding: 10px 15px;      /* Свойство: внутренние поля */
  border-radius: 5px;      /* Свойство: скругление углов */
}

/* Псевдо-класс :hover для изменения стиля при наведении */
.nav-button:hover {
  background-color: #555;
}

Основы JavaScript

Что такое JavaScript?

JavaScript (JS) — это высокоуровневый язык программирования, который делает веб-страницы интерактивными. Он позволяет изменять HTML и CSS, реагировать на действия пользователя (клики, ввод текста) и обмениваться данными с сервером.

Ключевые концепции:

  • Переменные: Контейнеры для хранения данных (let, const, var).
  • Типы данных: String, Number, Boolean, Object, Array и др.
  • Функции: Блоки кода, которые можно вызывать многократно.
  • DOM-манипуляции: Изменение HTML-структуры и CSS-стилей "на лету" (document.getElementById(), element.classList.add()).
  • События: Реакция на действия пользователя (addEventListener('click', ...)).
  • Асинхронность: Выполнение операций (например, загрузка данных с сервера) без блокировки основного потока (async/await, Promises, fetch()).

Пример: Сделать кнопку кликабельной

<!-- HTML -->
<button id="myButton">Нажми меня</button>
<p id="message"></p>

<!-- JavaScript -->
<script>
  // 1. Находим элементы на странице
  const button = document.getElementById('myButton');
  const message = document.getElementById('message');

  // 2. Добавляем "слушателя" события 'click'
  button.addEventListener('click', function() {
    // 3. Этот код выполнится при клике
    message.textContent = 'Кнопка была нажата!';
  });
</script>

Работа с данными в Excel

Основы работы в Excel

Microsoft Excel — это мощный инструмент для работы с электронными таблицами. Он используется для хранения, организации, анализа и визуализации данных.

Ключевые функции:

  • Ячейки, строки, столбцы: Основа структуры данных.
  • Формулы: Начинаются со знака =. Позволяют выполнять вычисления (=SUM(A1:A10), =AVERAGE(B1:B5)).
  • Абсолютные и относительные ссылки: A1 (относительная) изменяется при копировании, $A$1 (абсолютная) — нет.
  • Функция ВПР (VLOOKUP): Позволяет искать значение в одном столбце таблицы и возвращать соответствующее значение из другого столбца.
  • Сводные таблицы (Pivot Tables): Интерактивный инструмент для быстрого суммирования, анализа и группировки больших объемов данных.
  • Условное форматирование: Автоматически изменяет внешний вид ячеек (цвет, шрифт) на основе заданных правил (например, "подсветить все значения > 100").
  • Диаграммы и графики: Визуализация данных для лучшего восприятия.

Пример: Функция ВПР (VLOOKUP)

=VLOOKUP(A2, Sheet2!$A:$B, 2, FALSE)

Расшифровка:

  1. A2 — Что ищем (например, артикул товара).
  2. Sheet2!$A:$B — Где ищем (в столбцах A и B на Листе 2).
  3. 2 — Какой столбец вернуть (вернуть значение из 2-го столбца, т.е. из B).
  4. FALSE — Требовать точного совпадения.

Основы Design / UX

Что такое UX и UI?

UI (User Interface) — Пользовательский Интерфейс. Это то, что пользователь видит: кнопки, цвета, шрифты, макет. UI отвечает за визуальную часть.
UX (User Experience) — Пользовательский Опыт. Это то, что пользователь чувствует при взаимодействии с продуктом: насколько легко им пользоваться, логична ли навигация, решает ли продукт задачу пользователя.

Ключевые этапы UX-дизайна:

  • Исследование (Research): Понимание целевой аудитории, ее потребностей и "болей" (интервью, опросы).
  • Проектирование (Design):
    • Wireframes (Вайрфреймы): Низкодетализированные "скелеты" интерфейса. Показывают структуру и расположение элементов.
    • Mockups (Макеты): Высокодетализированные, "пиксель-перфектные" изображения того, как будет выглядеть конечный продукт (UI-дизайн).
    • Прототипы (Prototypes): Интерактивные макеты, которые позволяют "прокликать" основной сценарий использования.
  • Тестирование (Testing): Юзабилити-тестирование, когда реальным пользователям дается прототип для проверки гипотез и выявления проблем.
  • Иерархия и Контраст: Важные элементы должны быть более заметными (за счет размера, цвета, расположения).
  • Доступность (Accessibility, a11y): Проектирование интерфейсов так, чтобы ими могли пользоваться люди с различными ограничениями (контрастность, поддержка клавиатуры, ARIA-атрибуты).
